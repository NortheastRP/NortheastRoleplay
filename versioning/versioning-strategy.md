# NERP Versioning Strategy

## 🎯 Overview

Northeast Roleplay (NERP) follows a structured and forward-thinking versioning system designed to support:

- Stable releases
- Continuous improvements
- Parallel development of future updates

This ensures the live server remains polished while future content is developed without disruption.

---

## 🔢 Version Structure

### 🛠️ Development Build
**Version:** `v1-dev`

- Active development version of NERP
- Used during internal building of systems (Phases 2 & 3)
- Not stable and subject to frequent changes
- Not intended for public access

---

### 🧪 Testing Build
**Version:** `v1-beta`

- First playable version for staff and department testing
- Used during Phase 4 (Testing & Optimization)
- Focus on:
  - Bug discovery
  - Performance testing
  - System validation
- Limited access (staff, testers, departments)

---

### 🚀 Release Build
**Version:** `nerp-v1`

- Official public launch version
- Fully stable and feature-complete (Version 1.0)
- Represents completion of all 5 development phases

---

## 🔧 Patch & Update Versions

After release, NERP will follow a structured update cycle:

### 📌 Patch Fixes
**Versions:** `1.001 – 1.099`

Includes:
- Bug fixes
- Performance improvements
- Minor balance changes
- Small system adjustments

---

### 🎉 Content Updates

- Small content drops tied to:
  - Holidays (e.g. Halloween, Christmas)
  - Community events
- May include:
  - Limited-time features
  - Themed content
  - Minor gameplay additions

---

## 🔮 Future Development (Version 2.0)

While Version 1 is live:

- Development of **`v2-dev`** will begin in parallel
- Focus on:
  - Major system overhauls
  - New large-scale features
  - Expanded roleplay mechanics

---

### 🧠 Version 2 Lifecycle

| Stage | Version |
|------|--------|
| Development | `v2-dev` |
| Testing | `v2-beta` |
| Release | `nerp-v2` |

---

## 📣 Roadmap Integration

- Plans for Version 2.0 will be added to the roadmap in a future update
- Community feedback will influence development direction
- Transparency will remain a priority

---

## 🧩 Development Philosophy

NERP prioritizes:

- Stability before expansion
- No pay-to-win systems
- Community-driven improvements
- High-quality roleplay experiences

---

## ✅ Summary

NERP uses a **dual-track development system**:

- **Version 1 (Live):** Maintained with patches and small updates
- **Version 2 (Development):** Built in the background for future release

This allows continuous improvement without sacrificing stability.
